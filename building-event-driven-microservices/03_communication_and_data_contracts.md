# 3장. 통신 및 데이터 규약
- 이벤트 기반 시스템에서 이벤트는 무슨 일이 일어났고 그 이유는 무엇인지 정확하게 기술하는 메시지이자, 통신의 기본단위이다.
## 3.1. 이벤트 기반 데이터 규약
- 명시적인 스키마를 규약으로 사용

## 3.2. 이벤트 포멧
- 엄격하게 정의된 포맷을 추천(thrift, Protocol Buffer, JSON)
- 단순 키/값 구조의 평범한 텍스트에 대한 고민
## 2.2. 이벤트 설계
- 스트림당 이벤트 정의는 하나만 사용
- 가장 좁은 범위의 데이터 타입 적용
  - 문자열을 이용해 숫자 값을 저장
  - 정수를 boolean으로 사용
  - 문자열을 enum으로 사용
- 이벤트는 하나의 목적만 갖도록
